# Git + CI Interview Task

## Goal
Practice Git workflows and understand how CI pipelines validate changes automatically.

## Learning objectives
- Use Git branches, commits, and pull requests effectively
- Understand how CI checks code quality and build status
- Learn how to troubleshoot a failing pipeline

## Task
1. Create a branch for your work.
2. Make a small change in the project.
3. Commit the change.
4. Push the branch.
5. Observe how a CI pipeline would run on the branch.

## Suggested experiment
- Add a small script or update the README.
- Use GitHub Actions or another CI tool to run a simple validation step.

## Interview questions
- What is the difference between `git merge` and `git rebase`?
- Why do teams use pull requests?
- How would you handle a failed CI pipeline?
- What is the purpose of branching strategies?

## Bonus challenge
Create a GitHub Actions workflow that runs:
- a simple shell command
- a Python syntax check
- or any lightweight validation
