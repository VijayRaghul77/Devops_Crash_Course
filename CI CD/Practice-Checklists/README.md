# CI/CD Practice Checklists

This file helps you prepare for interview practice by checking the key concepts you should be able to explain and demonstrate.

## GitHub Actions Checklist
- [ ] Explain what triggers the workflow
- [ ] Explain the purpose of jobs and steps
- [ ] Describe how `needs:` controls flow
- [ ] Explain how secrets are managed
- [ ] Describe how approvals for production are handled

## Azure DevOps Checklist
- [ ] Explain stages, jobs, and deployment jobs
- [ ] Describe how environment approvals work
- [ ] Explain variables and secure file usage
- [ ] Talk about artifact reuse and deployment flow

## GitLab CI Checklist
- [ ] Explain `.gitlab-ci.yml`
- [ ] Describe stage ordering and parallel jobs
- [ ] Explain how `rules` control execution
- [ ] Explain how artifacts are shared between jobs

## Jenkins Checklist
- [ ] Explain Jenkinsfile structure
- [ ] Describe declarative vs scripted pipeline
- [ ] Explain agents and stages
- [ ] Explain post actions and notifications

## Kubernetes Checklist
- [ ] Explain Deployment vs Service
- [ ] Explain rolling update strategy
- [ ] Describe probe usage for readiness and liveness
- [ ] Explain rollback strategy in Kubernetes

## Terraform Checklist
- [ ] Explain what Terraform does
- [ ] Explain `plan` vs `apply`
- [ ] Explain why remote state is important
- [ ] Explain how CI/CD validates infrastructure changes
